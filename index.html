Philosophy

I believe in thoughtful, elegant coding. Coming from a low-level system programming background, I value mindfulness of every aspect of a web-based application design: performances and appropriate engineering, before shine and mindless richness of features, are essential to ensure a satisfying experience for the end-user and a cost-effective workflow for the people I work with.

I am passionate about what I do. Since I put my hands on my first computer at the age of 13, I've been always drawn towards the beauty of writing code, and the pleasure of creating out of thin air things that solved problems in some clever ways or made people's lives easier or happier. I enjoy engineering and developing interactive web applications and I'm always looking for the next thing to learn.

Curiosity, the drive to solve new problems and to improve on old ones are core values to me.

My choices

Front-end web development requires a heterogeneous set of tools and programming languages that come in various shapes and sizes. Over the course of my career I hone my toolset, choosing the tools that best fit my needs and style.

I choose to structure web markup in HTML5, oftentimes using HTML5 Boilerplate as a scaffolding for my projects. Where applicable, I employ a preprocessing engine such as Jade to improve readability and speed.

I like to use preprocessing engines to generate my CSS3 stylesheets -- my favorite is Sass. I also base most of my works upon the Zurb Foundation framework, which offers a wide and interesting mixin library or, occasionally, upon Twitter Bootstrap.

When developing Javascript, I tend to stick to native code as much as I can because indulgence could result in poor performances. Structuring code and developing robust interactive applications requires a certain amount of work, though, so frameworks are important, too. I like to modularize code according to the AMD standard, deployed using Require.js. Scaling upwards in terms of complexity for a web application, I employ MVC-based frameworks. When the tasks are lightweight, my framework of choice is Backbone.js, with the utility library Underscore.js, which comprises a simple but useful templating engine, and the general-purpose jQuery library. When dealing with large, complex, mission critical web applications that require lots of bindings and where consistency is key, I like to rely upon Angular.js.

My favorite backend engine is Node.js combined with the Express framework. I have extensive PHP experience too, having worked on everything ranging from small, single-page forms to large and complex portals. I am experienced in using the CodeIgniter MVC framework. I am also experienced in using Java, both for stand-alone applications and for the web (Java EE6). Generally I prefer working with RESTful interfaces and letting the front-end handle everything relative to the presentation of the data.

I have worked extensively using the MySQL database but I'm cultivating interest towards different storage approaches, namely NoSQL databases such as MongoDB.

My favorite code editor is Vim and my VCS of choice is Git. I develop my code on Mac OS X but like to tinker with other UNIX-like operating systems such as Linux distributions or FreeBSD/NetBSD/OpenBSD.
